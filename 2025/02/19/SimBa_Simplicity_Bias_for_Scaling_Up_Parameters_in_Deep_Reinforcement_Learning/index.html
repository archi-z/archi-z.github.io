SimBa: Simplicity Bias for Scaling Up Parameters in Deep Reinforcement Learning

原文：SimBa: Simplicity Bias for Scaling Up Parameters in Deep Reinforcement Learning

在DRL领域，如何设计网络使其能够从扩展参数量获益。

这篇文章提供了一个切入视角——simplicity bias（简单性偏见）。

基于这个视角，设计了SimBa网络结构。

实验验证了SimBa网络结构可以对多种RL算法，多种游戏场景提升算法表现。

前置知识

扩展神经网络规模是当今CV和NLP领域发展的一个关键驱动力。扩展模型参数一方面提升了模型的拟合能力，另一方面也增加了对训练数据的过拟合风险。

但事实上，这种神经网络倾向于朝简单的，泛化性好的函数收敛。这种现象被归因于神经网络本身具有的简单性偏见（simplicity bias）——优化算法和网络模块引导高表达能力模型朝向简单，泛化性好的函数收敛。例如随机梯度下降、ReLU、层归一化、残差模块。这些要素影响了神经网络在初始时刻表征函数的类型，而初始时刻表征函数越简单，网络越可能收敛到简单函数。

这篇文章通过放大简单性偏见，引导模型朝向更简单，泛化性更好的函数收敛。更准确的说法，应该是避免模型过早拟合到局部最优的价值函数。

定义1（函数复杂度）：对函数进行离散傅里叶变换，利用傅里叶系数对频率进行加权求和——\(c(f)=\sum_{k=0}^{K}(\tilde{f}(k)\cdot k) / \sum_{k=0}^{K}\tilde{f}(k)\)。

\(k\)越大意味着频率越大。\(c(f)\)越大，函数高频成分越多，越复杂。反之亦然。

定义2（简单性偏见得分）：\(s(f)\approx\mathbb{E}_{\theta\sim\Theta_{0}}[\frac{1}{c(f_{\theta})}]\)。其中\(\Theta_{0}\)是初始化网络参数的分布

由于不容易控制变量，论文比较的是不同网络结构初始化时刻的简单性偏见。有研究表明，网络训练期间收敛到的函数的复杂程度和网络初始化时刻的复杂程度强相关。

SimBa网络

SimBa网络存在一条直接从输入到输出的线性路径，仅通过残差连接实现非线性

输入归一化模块

设计理念：避免不成比例的大值特征主导学习过程。

残差模块

设计理念：模型保持一种简单状态，除非需要非线性转化。

层归一化模块

设计理念：确保激活值尺度一致。
